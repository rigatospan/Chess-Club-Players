Chess Teams Manager

Local software to create/manage/update chess teams and players info of a local club automatically. It is comprised out of 7 total sections:

  1. Configure Players List (top-left): define the criteria for searching the players database.

  2. Create New Team (bottom-left): crete teams and set team's information.

  3. Players Database (top-middle): the club's players database to be searched/restricted.

  4. Created Teams Frames (bottom-middle): the teams compositions to be filled with players.

  5. Configure Database Columns (top-right): configure the players database/teams with additional columns.

  6. Move Players to Teams (middle-right): select the team to place the selected players.

  7. Update Database (bottom-right): create/update the players database.

and a menu bar. Also some other minor actions.

1. Configure Players List

On this panel you can define the searching criteria on the players database. There are six total criteria to impose:

  1. elo type and range: choose one of the elo types (national, std, rapid, blitz) and its minimum and maximum range.

  2. age: choose the age range of the players to be displayed.

  3. gender: choose the gender of the players, 'All', 'M' for males, 'F' for females.

  4. active players: when selected yes only the fide active players will be displayed, when no only the inactive.

  5. Eso fee paid: when selected yes only the players with paid eso fee will be displayed.

  6. Search by name: search a player/players by name, either with latin or greek characters. When some text is inserted in the box, only the players with names that start from that text will be displayed when applied.

There are two buttons on this section as well:

  1. Apply: when clicked will display the players based on the current insertions on the above fields. Of course all fields are combined to restrict the database, e.g. one can search all female, active players, with fide elo in range 1500-2000, aged between 10-20.

  2. Clear: when clicked it clears all entries to the fields and automatically reset the database to its initial display of all players.

2. Create New Team

On this panel you can create multiple teams to be fill with players from the database. There are 8 fields to insert information about the upcoming match:

  1. Team's Name: provide the name of the club's team, e.g. So Aigaleo Under 10 A.

  2. Opponent Team's Name: provide the name of the opponent club's team, e.g. So Peristeriou Under 10 B.

  3. Tournament: provide the name of the competition that the team takes part in, e.g. Eso Tournament Under 10 2023.

  4. Match Date: probide the date of the match to take place: e.g. 1/1/2030.

  5. Address: provide the address of the place where the match takes place.

  6. Round: provide the number of the round, e.g 7th round.

  7. Home Court: if selected yes then the team has home-court.

  8. Number of Boards: provide the number of players that the team is comprised of, e.g. 8 total players.

There are two butons on this panel:

  1. create: when clicked it will create a team frame with the current provided information on the above fields.

  2. clear: when clicked it will empty the entries to all of the above fields.

Notes:

  • Teams are identified by there NAMES. You CANNOT create two teams with the same name.

  • If a team's frame is already created but the provided information need to be adjusted, simple provide the team's name in the appropriate field and all adjusted information and then click apply. No new frame will be created but the information on the existing one will be updated.

3. Players Database

On this panel the information about the club's players are displayed. On the top there are a few general information about the Club's Name, total number of players and last time the players database was updated. The players informations are organized in a table-dataframe manner as rows and columns. The table is interactive in the following operations:

  • when click on the name of a column the rows will be sorted in ascended order of the column values. When clicked on the same column name the order is descended.

  • when double click on a players fide id value, the player's fide profile will be opened in the web browser.

  • you can select multiple players by pressing Ctrl + left click and place those players on a team via the middle-right panel.

  • you can add/remove a column to the table using the right-top panel.

  • you can adjust the width of each column if needed.

4. Created Teams Frame

On this panel the frames of the created teams are displayed with the provided information and with the spedified number of players to be filled. There are 3 sections on this panel:

  • on the top the team's information is displayed, with an additional field for any optional additional information to be entered, e.g. some useful telephone numbers for the upcoming match.

  • in the middle there is a similar database table with the players of this team.

  • in the bottom there are 5 buttons:

    1. Remove team: remove the created frame of the team

    2. Remove Selected Players: removes the currently selected players out of the team's composition.

    3. Move Player Up/Down: when pressed move the currently selected player up/down in the team's composition order.

    4. Export to pdf: when pressed creates a pdf with the current state of the team in the same folder as the executable file and with name the name of the team.

Notes:

  • You can switch between the teams frames.

  • You can fill a team with players using the middle-right panel appropriately.

  • You cannot place the same player twice on the same team.

  • You could place the same player twice on different teams, however there is a warning for that to avoid confusions.

  • The order of the players and any restrictions on the teams composition is to be handled manualy by the user. For example the order of the players may be determined by elo, there might be elo and age restrictions per board number etc.

  • When export to a pdf, the columns currently present on the table will be present on the pdf file. If some additional columns are needed, impose those on the team's table by using the top-right panel appropriately.

5. Configure Table Columns

Use this panel to add/remove and configure which columns to display and their order, in both the players database and on each team frame separetely. The columns and order to be applied is presented on the list box and configured appropriately by the operations of this panel.

There are 8 widgets on this panel:

  1. Columns to show: when clicked the list of all avaliable columns is displayed. Select the column you want to add or remove.

  2. Add/Remove Buttons: when pressed adds/removes the selected column to the list of columns displayed below. The added column is always placed last.

  3. Move up/down Buttons: when a column name is selected from the list of columns it moves that column up/down the list to the desired order.

  4. Reset Button: reset the columns list to the default columns.

  5. Clear Button: emptys the columns list.

  6. Columns list: the list of columns and order to be applyed to the database or a team's frame.

  7. Teams list: the list of teams created and the database. Select the one you want to configure its columns. When a team is created is added to this list.

  8. Apply Columns Button: apply the current list and order of columns to the selected team or to the database.

6. Fill Teams with Selected Players

Use this panel to place players from the database to a team's composition with the following two widges:

  1. Select Team list: select a team from the created teams to place players to.

  2. Move Players to Team Button: When pressed the currently selected players from the database will be place to the selected team.

Notes:

  • Select one or multiple players from the database by pressing (hold) Ctrl + left click, then select the desired team to place those players and press the Move Players to Team button.

  • A player cannot be twice on the same team, but it CAN be on two different teams.

  • the total number of selected players cannot excide the number of players of the team.

7. Update Database

On this panel you can create a club's players database combining both national and internation information about the players. using the following:

  • Enter Club's Eso URL: enter the Eso club's URL, e.g. https://chesstu.be/eso/club/01151

  • Create/Update Database: when pressed will automatically create/update the database of the provided url's club. Ensure that there is an internet connection.

  • Time-elapsed bar: indicates the time remaining for the database to update.

Notes:

  • Database: the programm will create a database with the players information from BOTH the eso and fide sites as a .pkl file on the same folder location as the program's executable file and a .json file with metadata of the Club. Keep those files on the same folder as the executable file.

  • If the Club's database has already been created its URL is listed on the box and pressing the Update button will update the current database to the latest player's information.

  • At the begining of each month a reminder window is displayed to the user to update the database to fetch the latest elo.

  • If multiple clubs database exist on the folder the program will automatially display the latest updated one.

8. Menu Bar

At the moment there are two sections on the menu bar:

  • Documentation: open the documentation of the application on the Web browser.

  • Import Database: open a file dialog to select a .pkl database to display in the application window.

9. Minor Actions

At the moment there are the additional actions:

  • Save teams upon closing: when closing the application after creating some teams, a save option appears. If selected 'yes' then the current teams will be saved as files in the created_teams_data folder on the same directory as the executable file. When opening the application again the teams that were saved last time will be displayed. All teams saved previously will be erased, only the last saved teams are saved.
